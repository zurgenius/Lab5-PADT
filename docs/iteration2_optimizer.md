# Итерация 2: Visitor и QueryOptimizer

> Документ фиксирует состояние второй итерации. Актуальное описание завершенного интерфейса находится в [ui.md](ui.md).

## Цель

Вторая итерация добавляет оптимизатор запросов поверх существующих DSL и AST. Оптимизатор не исполняет запросы, а строит новое дерево выражения и сохраняет отчет о примененных правилах.

## Архитектура

Код разделен на два слоя:

- `include/visitor/` - базовый `Visitor` и утилиты анализа AST;
- `include/query_optimizer/` - `QueryOptimizer`, `OptimizationResult` и `OptimizationReport`.

`QueryOptimizer::optimize(root)` возвращает результат с тремя частями:

- `original()` - исходный `ExprPtr`;
- `optimized()` - новый оптимизированный `ExprPtr`;
- `report()` - примененные правила и предупреждения.

Исходное дерево не изменяется. Все публичные списки передаются через `Sequence<std::string>`, а внутренние накопители используют `MutableArraySequence<std::string>`.

## Правила

Реализованы консервативные правила:

- `Select(Project(x)) -> Project(Select(x))`, если условие использует только сохраненные колонки;
- `Select(Join(left, right)) -> Join(Select(left), right)`, если условие относится только к левой стороне;
- `Select(Join(left, right)) -> Join(left, Select(right))`, если условие относится только к правой стороне;
- `Project(Project(x)) -> Project(x)` с внешним списком колонок;
- `Project(Join(left, right)) -> Project(Join(Project(left), Project(right)))`, если колонки однозначно делятся по сторонам join.

Join-ключи всегда сохраняются при projection pruning. Если колонка неоднозначна, правило не применяется.

## Условия

Оптимизатор анализирует только структурированные условия:

```cpp
ra::Gt("users.age", ra::Literal(20))
ra::Eq("users.id", ra::Column("orders.user_id"))
```

Raw-условия вроде `"users.age > 20"` не разбираются. Они остаются на месте, а в отчете появляется предупреждение.

## UI

Вкладка `DSL/AST` показывает исходный запрос. Вкладка `Optimizer` для того же примера показывает:

- RA-нотацию до и после;
- AST до и после;
- отчет с примененными правилами и предупреждениями.

Добавлены демонстрационные примеры для push-down selection и projection pruning. В состоянии второй итерации вкладка `Backends` оставалась точкой расширения; ее текущее поведение описано в [ui.md](ui.md).
