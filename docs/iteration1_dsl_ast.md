# Итерация 1: DSL и AST

> Документ фиксирует состояние первой итерации. Актуальное описание завершенного интерфейса находится в [ui.md](ui.md).

## Цель

Первый шаг реализует чистое дерево выражений реляционной алгебры и C++ DSL для его построения. Дерево пока не исполняет запросы и не оптимизирует их: оно хранит структуру запроса, схему результата и описание условий.

## Архитектура

Код первого шага разложен по слоям:

- `include/ast/` - AST-узлы, базовое выражение и printer;
- `include/dsl/` - условия и функции DSL;
- `include/common/` - общие утилиты для `Sequence<std::string>` и описания источников таблиц;
- `include/visitor/` - общий visitor-интерфейс и утилиты обхода для следующих итераций;
- `include/*/detail/` - шаблонные реализации core/streams типов;
- `src/` - нетемплейтные реализации и точка входа demo.

Базовый тип `RelationalExpr` задает общий интерфейс AST-узла:

- `kind()` возвращает тип операции;
- `schema()` возвращает схему результата;
- `accept(Visitor&)` подключает обход дерева.

Реализованные узлы:

- `TableExpr` - отношение-источник с именем и схемой;
- `SelectExpr` - выборка строк по описанному условию;
- `ProjectExpr` - выбор колонок;
- `JoinExpr` - соединение двух выражений по паре колонок;
- `RenameExpr` - алиас выражения для будущей работы с неоднозначными именами.

Дочерние узлы хранятся через `std::shared_ptr<RelationalExpr>`. Публичный API работает через `Sequence<std::string>`, а конкретное хранение внутри узлов использует `MutableArraySequence<std::string>` из `include/core`; `std::vector` в RA-слое не используется.

## DSL

Доступны два стиля записи.

```cpp
#include "dsl/query.h"
```

Функциональный стиль:

```cpp
auto users = ra::Table("users", {"id", "name", "age"});
auto query = ra::Pi({"name"}, ra::Sigma("age > 20", users));
```

Pipe-стиль:

```cpp
auto query = ra::Table("users", {"id", "name", "age"}) |
             ra::Sigma(ra::Gt("age", ra::Literal(20))) |
             ra::Pi({"name"});
```

Условия первого шага являются описанием предиката. Они нужны для печати, будущего анализа оптимизатором и будущих backend-переводчиков.

## Вывод дерева

`AstTreePrinter` печатает дерево AST с отступами, а `RelationalAlgebraPrinter` выводит компактную RA-нотацию:

```text
pi[name](sigma[age > 20](users))
```

## ImGui Demo

Графическое demo запускается командой:

```bash
make run
```

Вкладка `DSL/AST` показывает готовые запросы, DSL-текст, RA-нотацию, дерево AST и схему результата. В состоянии первой итерации вкладки `Optimizer` и `Backends` были точками расширения; их текущее поведение описано в [ui.md](ui.md).

## Ограничения первого шага

- CSV и SQLite не подключены.
- `open/next/close` и реальное исполнение строк не реализуются.
- Оптимизатор реализуется отдельным слоем во второй итерации.
- `ProjectExpr` проверяет наличие колонок в схеме дочернего выражения и выбрасывает исключение при неизвестной колонке.
